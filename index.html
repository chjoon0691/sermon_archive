유튜브 설교 주소를 넣으면 아카이브 자료를 생성합니다

설교 영상 주소를 입력하면 설교자, 제목, 날짜, 본문, 오타 수정본, 요약, 대지, 예화, 주제 색인을 생성하는 화면입니다. 정적 GitHub Pages에서는 API 비밀키를 보관할 수 없으므로, 실제 생성은 별도 백엔드 API와 연결해야 합니다.

이 파일은 화면과 연결 로직을 모두 포함합니다. 실제 작동을 위해서는 아래 백엔드 API 주소에 설교 처리 서버 주소를 넣어야 합니다. API가 없으면 데모 결과가 표시됩니다.

7단계 자막·전사·교정·분석·예화·색인·저장
JSON GitHub 저장용 데이터 다운로드 가능

1. 설교 영상 주소 입력

유튜브 주소를 넣고 생성 버튼을 누르세요. 백엔드 API가 연결되어 있으면 실제 분석 결과를 받고, 연결되어 있지 않으면 화면 테스트용 데모 결과가 생성됩니다.

권장 백엔드 응답 형식은 이 파일 아래쪽 JavaScript의 expectedResponseShape 주석을 참고하세요.

2. 처리 진행 상황

1
주소 확인유튜브 주소 형식을 확인합니다.
2
영상 정보 수집제목, 채널, 날짜, 설명란을 가져옵니다.
3
자막·전사 확보자막이 있으면 사용하고, 없으면 전사를 요청합니다.
4
오타 수정원문 흐름을 보존하며 오타와 띄어쓰기를 정리합니다.
5
설교 분석요점, 대지, 핵심 메시지를 정리합니다.
6
예화 추출예화와 연결 가능한 주제를 분리합니다.
7
아카이브 저장브라우저 저장소와 JSON 파일로 보관합니다.
아직 생성된 결과가 없습니다.
위에 유튜브 주소를 입력하고 생성 버튼을 눌러주세요.

3. 생성된 설교 아카이브

이 목록은 현재 브라우저의 LocalStorage에 저장됩니다. GitHub 저장소에 반영하려면 JSON을 다운로드해 data/sermons.json에 합치면 됩니다.

GitHub Pages는 정적 사이트이므로 OpenAI API 키, YouTube 처리 권한, 음성 전사 작업은 프론트엔드에 넣으면 안 됩니다. 실제 운영에서는 Vercel, Render, Cloudflare Workers, Railway, 개인 서버 등 별도 백엔드에서 처리한 뒤 이 화면으로 결과를 반환하게 하세요.
